# i18n Design: Selector de Idioma

**Fecha:** 2026-04-08
**Idiomas:** es (raiz), en (/en), fr (/fr), zh (/zh)

## Arquitectura

### Rutas
- Espanol: `/`, `/actividades/tour-espiritu-santo`, etc.
- Ingles: `/en`, `/en/actividades/tour-espiritu-santo`, etc.
- Frances: `/fr`, `/fr/actividades/tour-espiritu-santo`, etc.
- Chino: `/zh`, `/zh/actividades/tour-espiritu-santo`, etc.

### Estructura de archivos de traduccion
```
locales/
  es.json  -- espanol (base)
  en.json  -- ingles
  fr.json  -- frances
  zh.json  -- chino simplificado
```

Cada JSON organizado por secciones: nav, hero, adventures, testimonials, expeditions, faq, footer, etc.

### Helper i18n
`lib/i18n.ts` exporta `getDictionary(locale)` que carga el JSON correspondiente. Sin librerias externas.

### Middleware
`middleware.ts` en la raiz:
- Parsea el pathname para extraer locale
- Si no hay prefijo, asume `es`
- Reescribe internamente a `app/[locale]/...`

### Selector de idioma
`components/LanguageSelector.tsx`:
- Dropdown en el Header con ES, EN, FR, ZH
- Al seleccionar, navega a la misma ruta en el otro idioma
- Muestra el idioma actual resaltado

### SEO
- `<html lang>` dinamico segun locale
- hreflang tags para las 4 versiones
- sitemap.ts incluye URLs de los 4 idiomas
- Metadata traducido por idioma

### Migracion
- Mover todas las rutas actuales dentro de `app/[locale]/`
- El layout raiz (`app/layout.tsx`) redirige a `app/[locale]/layout.tsx`
- Traducciones iniciales generadas automaticamente, para revision posterior

## Locales soportados
- `es` -- Espanol (default, sin prefijo URL)
- `en` -- English
- `fr` -- Francais
- `zh` -- Zhongwen simplificado (zh-CN)
